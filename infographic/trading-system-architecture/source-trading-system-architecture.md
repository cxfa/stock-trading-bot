# A股智能交易系统 v2.0 架构全景

## 系统概述
A股模拟交易系统，初始资金 ¥1,000,000。通过多因子策略、LLM辩论、技术分析自动运行。

## 核心模块（6大子系统）

### 🧠 交易策略 (trading_strategy.py) — 大脑
- 多因子决策引擎：大盘指数 + 新闻情绪 + 技术指标 + 风控 + 市场状态
- 盘中快速调整：每30分钟，<1秒，无LLM
- 复盘全面调整：每日收盘后，深度分析，调用LLM
- 输出 Signal 信号（买入/卖出/持有 + 信心度 + 原因）

### 👁️ 盘中监控 (monitor_v2.py) — 眼睛
- 每1分钟循环：获取实时行情 → 调用策略判断 → 执行交易
- 每30分钟：飞书快报 + 策略快速调整
- 交易触发：飞书立即通知

### 🤲 交易执行器 (trade_executor.py) — 双手
- 纯买卖执行，不做判断
- 原子文件锁（fcntl），防止TOCTOU竞态
- 佣金计算、T+1冻结、持仓均价、交易日志

### 🔍 LLM复盘引擎 (llm_review_engine.py) — 反思
- 系统健康检查（数据完整性、参数版本、日志、资产一致性）
- Sharpe比率、胜率、最大回撤等统计
- LLM深度分析（调用OpenClaw配置的模型）
- 生成买入计划 buy_plan.json
- 全面策略调整 strategy_params.json

### 🔭 选股系统 (stock_discovery.py) — 侦察兵
- 7源筛选：涨幅榜、连涨、机构、板块、技术、AI基建、BaoStock
- 质量评分 + 复盘评分加成（+10/-15）
- 多日跟踪加成（+15/-5，RSI/动量/成交量）
- 只提供候选，不参与交易策略

### ⏰ 调度器 (scheduler_daemon.py) — 心脏
- 统一调度，替代cron
- 交易日历（节假日+周末自动跳过）
- 管理 monitor_v2 子进程生命周期

## 每日运行周期

### 09:20 盘前准备
获取市场情绪

### 09:30-11:30 / 13:00-15:00 盘中
monitor_v2 每1分钟循环运行
每30分钟：策略快速调整 + 飞书快报

### 15:05 收盘报告
生成日度交易总结

### 15:30 LLM复盘
系统健康检查 → LLM深度分析 → 全面策略调整 → 买入计划生成

### 15:45 盘后选股
读取复盘输出 → 多日跟踪更新 → 7源选股 → 候选股池

## AI与系统的关系

### 🤖 AI的职责（非实时，离线）
1. 环境搭建：安装依赖、配置环境、启动daemon
2. 复盘分析：LLM深度分析交易结果和盈亏原因
3. 参数调优：根据复盘结果调整 strategy_params.json
4. 代码改进：修复bug、实现改进假设
5. 系统健康检查：验证数据完整性和进程状态

### ⚙️ 系统自主运行（实时，无AI）
1. 调度器自动管理生命周期
2. monitor_v2 每分钟循环
3. 策略自主判断买卖
4. 执行器自动执行交易
5. 飞书自动通知

## 核心数据流
选股 → discovered_stocks.json → 复盘 → buy_plan.json → monitor_v2 → 策略判断 → 执行器
复盘 → review_output.json → 选股评分加成
复盘 → strategy_params.json → 策略参数更新
盘中 → strategy_state.json → 盘中快调记录
